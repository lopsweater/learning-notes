## Agent 任务

```
你是一个游戏引擎分析专家。请对以下游戏引擎进行深度分析：

## 引擎信息

- 引擎名称: [ENGINE_NAME]
- 源码路径: [SOURCE_PATH]
- 版本控制: [SVN / Git / None]
- 主要语言: [C++ / C# / Other]

## 任务目标

按照以下阶段对引擎进行系统性分析，输出文档到 `engine-doc-template/` 目录。

---

## 阶段一：架构宏观分析

**输出文件**: `engine-doc-template/architecture/README.md`

**任务**:

1. 分析源码目录结构，识别核心模块
   - 列出顶层目录及其职责
   - 识别模块间的依赖方向

2. 绘制架构层次图（使用 ASCII 图）
   - 展示核心子系统及其关系
   - 标注数据流向
   - 标注模块边界

3. 识别并分析核心子系统：
   - 渲染系统（RHI、渲染管线、材质系统）
   - 资源管理系统（加载、缓存、生命周期）
   - ECS/场景系统（实体、组件、系统）
   - 脚本系统（语言、绑定、生命周期）
   - 物理系统（引擎集成、碰撞检测）
   - 音频系统（音频处理、3D音频）
   - 其他核心系统

4. 总结设计原则和技术选型

完成后汇报进度，继续下一阶段。

---

## 阶段二：模块深度分析

**输出目录**: `engine-doc-template/modules/`

**任务**:

1. 为每个核心模块创建文档：`modules/[模块名]/README.md`

2. 每个模块文档包含：
   - **概述**: 模块职责和定位
   - **架构图**: 模块内部结构
   - **核心类与接口**:
     ```markdown
     ### [类名]
     **职责**: ...
     **文件**: path/to/file.cpp
     **关键方法**:
     - `Method1()`: 说明
     - `Method2()`: 说明
     ```
   - **数据结构**: 关键数据结构说明
   - **设计模式**: 使用的模式及其目的
   - **依赖关系**: 依赖哪些模块，被哪些模块依赖
   - **线程模型**: 单线程/多线程，同步机制
   - **性能考量**: 性能关键点和优化策略

3. 提取关键代码片段（带文件路径和行号）

完成后汇报进度，继续下一阶段。

---

## 阶段三：版本历史追溯

**输出目录**: `engine-doc-template/changelog/`

**任务** (如果有版本控制):

1. 创建 `changelog/evolution.md`:
   - 解析提交历史
   - 按时间线整理重大变更
   - 识别关键功能的演进过程
   - 分析版本里程碑

2. 创建 `changelog/contributors.md` (可选):
   - 开发者贡献分布
   - 核心贡献者及其负责模块

如果无版本控制，跳过此阶段。

---

## 阶段四：关键概念文档化

**输出文件**:
- `engine-doc-template/guides/` - 开发者指南
- `engine-doc-template/api-reference/` - API 参考
- `engine-doc-template/glossary.md` - 术语表

**任务**:

1. 创建 `glossary.md`:
   - 列出引擎特有的概念和术语
   - 每个概念给出简洁定义和关联概念

2. 创建 `guides/[关键概念].md`:
   - 选择 5-10 个关键概念（如渲染管线、资源生命周期、ECS 架构等）
   - 每个概念详细解释：
     - 概念定义
     - 在引擎中的实现
     - 使用示例
     - 最佳实践
     - 常见问题

3. 创建 `api-reference/README.md`:
   - 核心类索引
   - 按模块组织的 API 列表
   - 关键 API 的使用示例

---

## 输出规范

### 文档格式
- Markdown 格式

- ASCII 图表展示架构

### 重点标记
- ★ 必须理解
- ★★★ 核心重点
- ⚠️ 注意事项
- 💡 设计亮点

### 代码引用格式
```markdown

// 行号: 123-145
// 说明: 简要描述

代码片段...
```

3. 循序渐进，每完成一个阶段汇报进度

## 使用说明

1. 复制上面的完整提示词
2. 替换占位符：
   - `[ENGINE_NAME]` → 引擎名称
   - `[SOURCE_PATH]` → 源码路径
   - `[SVN / Git / None]` → 版本控制类型
   - `[C++ / C# / Other]` → 主要语言
3. 发送给 agent 执行
