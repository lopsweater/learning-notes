## 分析任务

### 阶段一：架构宏观分析

**目标**: 建立引擎的全局认知

**输出**: 填充 `engine-doc-template/architecture/README.md`

**任务清单**:
1. 分析源码目录结构，识别核心模块
2. 绘制架构层次图（使用 ASCII 或 Mermaid）
3. 识别核心子系统：
   - 渲染系统
   - 资源管理系统
   - ECS/场景系统
   - 脚本系统
   - 物理系统
   - 音频系统
   - 其他核心系统
4. 总结设计原则和技术选型

### 阶段二：模块深度分析

**目标**: 深入理解各模块实现细节

**输出**: 在 `engine-doc-template/modules/` 下创建各模块文档

**任务清单**:
1. 识别模块边界和依赖关系
2. 分析每个模块的：
   - 核心类和接口
   - 数据结构和算法
   - 线程模型
   - 性能考量
3. 提取关键代码片段和设计模式

### 阶段三：版本历史追溯

**目标**: 理解引擎的演进历程

**输出**: 在 `engine-doc-template/changelog/` 下创建演进文档

**任务清单**:
1. 解析版本控制提交历史
2. 按时间线整理重大变更
3. 识别关键功能的演进过程
4. 分析开发者贡献分布（如有需要）

### 阶段四：关键概念文档化

**目标**: 建立引擎知识体系

**输出**: 
- `engine-doc-template/guides/` - 开发者指南
- `engine-doc-template/api-reference/` - API 参考
- `engine-doc-template/glossary.md` - 术语表

**任务清单**:
1. 提取并解释核心概念（如渲染管线、资源生命周期等）
2. 编写关键 API 的使用示例
3. 建立概念间的关联关系图
4. 编写学习路径建议

## 输出要求

### 文档格式
- 使用 Markdown 格式
- 代码块指定语言高亮
- 使用 ASCII 或 Mermaid 绘制图表
- 使用表格进行对比分析

### 重点标记
- ★ 必须理解
- ★★★ 核心重点
- ⚠️ 注意事项
- 💡 设计亮点

### 代码引用规范
```

```

## 约束条件

1. **不修改源码**: 只读取分析，不修改任何源码文件
2. **文档位置**: 所有输出文档放在 `engine-doc-template/` 目录下
3. **循序渐进**: 先完成架构分析，再深入模块细节
4. **实用导向**: 重点关注开发者在实际使用中需要了解的内容
