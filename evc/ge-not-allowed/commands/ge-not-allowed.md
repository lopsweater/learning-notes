# /ge-not-allowed - 智能违规检查与规则学习

自动检查当前上下文违规情况，提示 Claude Code 修改，并将违规事项记录为禁止规则。

---

## 用法

```
/ge-not-allowed                # 检查当前上下文违规情况
/ge-not-allowed list           # 查看所有规则
/ge-not-allowed list <分类>    # 查看指定分类的规则
/ge-not-allowed clear          # 清空规则文件
```

---

## 核心功能

### 1. 智能违规检查 (`/ge-not-allowed`)

**执行流程**:

```
1. 读取规则文件 (.claude/rules/ge-not-allowed.md)
2. 分析当前对话上下文
   ├─ 检查最近的命令
   ├─ 检查文件操作
   ├─ 检查 git 操作
   └─ 检查代码修改
3. 匹配规则关键词
4. 生成违规报告
5. 提供修改建议
6. 询问是否记录为新规则
```

```
   3. 向用户展示变更内容
```

## 自动学习机制

### 工作流程

```
┌─────────────────────────────────────────────────────────┐
│          ge-not-allowed 自动学习流程                     │
└─────────────────────────────────────────────────────────┘

1. 用户执行操作
   │
   ▼
2. Claude Code 自动调用 /ge-not-allowed 检查
   │
   ├─ 无违规 → 继续执行
   │
   └─ 发现违规 → 显示违规报告
                  │
                  ▼
               3. 提示修改建议
                  │
                  ▼
               4. 询问用户是否记录规则
                  │
                  ├─ 用户同意 → 自动添加到规则文件
                  │            │
                  │            └─ 更新统计信息
                  │
                  └─ 用户拒绝 → 仅提示，不记录
```

### 自动记录格式

当用户确认记录新规则时，系统会：

1. **分析违规类型**
   - 判断属于哪个分类
   - 如果是新类型，创建新分类

2. **生成规则条目**
   ```markdown
   - ❌ <违规操作描述>
     📍 添加时间: <时间戳>
     📍 触发场景: <具体场景>
   ```

3. **写入规则文件**
   - 追加到对应分类
   - 更新统计数据

## 配置选项

在 `.claude/config.json` 中：

```json
{
  "ge-not-allowed": {
    "autoCheck": true,
    "showSuggestions": true,
    "askToRecord": true,
    "logFile": ".claude/ge-not-allowed.log",
    "rulesFile": ".claude/rules/ge-not-allowed.md"
  }
}
```

### 配置说明

| 选项 | 默认值 | 说明 |
|------|--------|------|
| `autoCheck` | true | 自动检查每次操作 |
| `showSuggestions` | true | 显示修改建议 |
| `askToRecord` | true | 询问是否记录新规则 |
| `logFile` | null | 日志文件路径 |
| `rulesFile` | 默认路径 | 规则文件路径 |

```markdown
- 新发现的违规模式，询问用户是否记录

规则文件位置: `.claude/rules/ge-not-allowed.md`
```
