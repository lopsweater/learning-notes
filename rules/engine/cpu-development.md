# CPU 侧开发流程

> 此文件描述游戏引擎 CPU 侧组件的开发流程。

## 适用范围

- 数学库（向量、矩阵、四元数）
- 内存分配器（线性、池、栈）
- 字符串处理
- 文件系统
- Asset 解析
- ECS 系统

## 开发流程

### 1. 规划阶段

- 确认功能需求
- 设计 API 接口
- 考虑 SIMD 优化需求
- 考虑多线程安全需求

### 2. TDD 开发

```
红 → 编写失败测试
绿 → 实现最小代码
重构 → 优化代码
验证 → 覆盖率 + 性能基准
```

### 3. 性能验证

#### 数学库性能基准

| 操作 | 目标 | 测试方式 |
|------|------|----------|
| 向量加法 | <10ns | Benchmark |
| 向量点积 | <10ns | Benchmark |
| 矩阵乘法 | <100ns | Benchmark |
| 四元数旋转 | <50ns | Benchmark |

#### 内存分配器性能基准

| 操作 | 目标 | 测试方式 |
|------|------|----------|
| 线性分配 | <100ns | Benchmark |
| 池分配 | <500ns | Benchmark |
| 释放 | <50ns | Benchmark |

### 4. SIMD 优化

#### 对齐要求

| 类型 | 对齐 |
|------|------|
| Vec3 (SSE) | 16 字节 |
| Vec4 (SSE) | 16 字节 |
| Mat4 (SSE) | 16 字节 |
| Vec8 (AVX) | 32 字节 |

#### SIMD 检查清单

- [ ] 使用 `_mm_load_ps` 对齐加载
- [ ] 避免未对齐内存访问
- [ ] 使用 `alignas` 对齐结构体
- [ ] 验证性能提升（至少 2x）

### 5. 多线程安全

#### 检查项

- [ ] 确认是否需要线程安全
- [ ] 使用 `std::mutex` 或原子操作
- [ ] 编写并发测试
- [ ] 使用 ThreadSanitizer 验证

```bash
cmake -DCMAKE_CXX_FLAGS="-fsanitize=thread" ..
```

### 6. 内存安全

#### 检查项

- [ ] 无内存泄漏
- [ ] 无悬垂指针
- [ ] 无缓冲区溢出
- [ ] 使用 AddressSanitizer 验证

```bash
cmake -DCMAKE_CXX_FLAGS="-fsanitize=address,undefined" ..
```

### 7. 代码审查

#### CPU 侧特有审查项

- [ ] SIMD 对齐正确
- [ ] 内存分配无泄漏
- [ ] 无数据竞争
- [ ] 性能达标
- [ ] 覆盖率 80%+
