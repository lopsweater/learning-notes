# 分析流程检查清单

> 用于跟踪游戏引擎分析进度

## 准备阶段

- [ ] 确认引擎源码路径存在
- [ ] 确认有读取权限
- [ ] 创建输出目录 `engine-doc-template/`
- [ ] 准备分析工具（Doxygen、Graphviz 等）

## 阶段一：架构宏观分析

- [ ] 分析源码目录结构
- [ ] 列出顶层目录及其职责
- [ ] 识别模块依赖方向
- [ ] 绘制架构层次图
- [ ] 识别核心子系统
  - [ ] 渲染系统
  - [ ] 资源管理系统
  - [ ] ECS/场景系统
  - [ ] 脚本系统
  - [ ] 物理系统
  - [ ] 音频系统
- [ ] 总结设计原则和技术选型
- [ ] 输出 `architecture/README.md`

## 阶段二：模块深度分析

- [ ] 确定要分析的模块列表
- [ ] 为每个模块创建文档
  - [ ] 模块概述
  - [ ] 架构图
  - [ ] 核心类与接口
  - [ ] 数据结构
  - [ ] 设计模式
  - [ ] 依赖关系
  - [ ] 线程模型
  - [ ] 性能考量
- [ ] 提取关键代码片段
- [ ] 输出 `modules/[模块名]/README.md`

## 阶段三：版本历史追溯

- [ ] 检查是否有版本控制系统
- [ ] 解析提交历史
- [ ] 按时间线整理重大变更
- [ ] 识别关键功能演进
- [ ] 分析版本里程碑
- [ ] （可选）分析贡献者分布
- [ ] 输出 `changelog/evolution.md`

## 阶段四：关键概念文档化

- [ ] 创建术语表
- [ ] 选择 5-10 个关键概念
- [ ] 为每个概念编写详细指南
- [ ] 创建 API 索引
- [ ] 输出 `glossary.md`
- [ ] 输出 `guides/[概念].md`
- [ ] 输出 `api-reference/README.md`

## 完成检查

- [ ] 所有文档格式正确（Markdown）
- [ ] 代码片段包含文件路径和行号
- [ ] 架构图清晰可读
- [ ] 关键概念有示例代码
- [ ] 文档交叉引用正确
