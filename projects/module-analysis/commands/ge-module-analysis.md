# /ge-module-analysis - 游戏引擎模块分析

对游戏引擎源码进行系统化深度分析，生成架构文档、模块剖析、版本历史和概念指南。

## 使用方法

```bash
/ge-module-analysis                                    # 分析当前目录引擎
/ge-module-analysis --engine UnrealEngine --path /root/UnrealEngine
/ge-module-analysis --engine Godot --phases 1,2       # 只执行阶段1和2
/ge-module-analysis --output ./docs                   # 自定义输出目录
/ge-module-analysis --focus rendering,physics         # 重点关注模块
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--engine` | 引擎名称 (UnrealEngine/Unity/Godot/Custom) | 自动检测 |
| `--path` | 源码路径 | 当前目录 |
| `--phases` | 分析阶段 (1,2,3,4) | 全部执行 |
| `--output` | 输出目录 | `./engine-doc-template` |
| `--focus` | 重点模块 | 所有核心模块 |
| `--lang` | 主要语言 (cpp/csharp/rust) | 自动检测 |

## 功能说明

### 四阶段分析流程

1. **阶段一：架构宏观分析** (10-20分钟)
   - 分析源码目录结构
   - 识别核心模块和依赖关系
   - 绘制架构层次图
   - 识别核心子系统

2. **阶段二：模块深度分析** (20-40分钟)
   - 深入剖析各子系统实现
   - 提取核心类和接口
   - 分析设计模式
   - 提取关键代码片段

3. **阶段三：版本历史追溯** (5-15分钟)
   - 解析 Git/SVN 提交历史
   - 按时间线整理重大变更
   - 识别功能演进过程

4. **阶段四：关键概念文档化** (15-25分钟)
   - 创建术语表
   - 编写开发者指南
   - 生成 API 参考

## 分析步骤

### 步骤 1：准备环境

```bash
# 检查源码路径是否存在
if [ ! -d "$ENGINE_PATH" ]; then
  echo "Error: Engine source path not found: $ENGINE_PATH"
  exit 1
fi

# 检测版本控制系统
if [ -d "$ENGINE_PATH/.git" ]; then
  VCS="git"
elif [ -d "$ENGINE_PATH/.svn" ]; then
  VCS="svn"
else
  VCS="none"
fi

# 检测主要语言
CPP_COUNT=$(find "$ENGINE_PATH" -name "*.cpp" | wc -l)
CS_COUNT=$(find "$ENGINE_PATH" -name "*.cs" | wc -l)

if [ $CPP_COUNT -gt $CS_COUNT ]; then
  LANG="cpp"
else
  LANG="csharp"
fi

# 创建输出目录
mkdir -p "$OUTPUT_DIR"/{architecture,modules,guides,api-reference,changelog}
```

### 步骤 2：架构分析（阶段一）

```bash
# 分析目录结构
echo "## Analyzing directory structure..."
find "$ENGINE_PATH" -maxdepth 1 -type d | sort

# 统计各目录文件数量
echo "## File distribution by directory..."
find "$ENGINE_PATH" -name "*.cpp" -o -name "*.h" | cut -d'/' -f2 | sort | uniq -c | sort -rn | head -20

# 查找构建系统
echo "## Build system detection..."
find "$ENGINE_PATH" -name "CMakeLists.txt" -o -name "*.sln" -o -name "Makefile" | head -10

# 生成依赖图（如果使用 CMake）
if [ -f "$ENGINE_PATH/CMakeLists.txt" ]; then
  cmake --graphviz="$OUTPUT_DIR/architecture/deps.dot" "$ENGINE_PATH"
  dot -Tpng "$OUTPUT_DIR/architecture/deps.dot" -o "$OUTPUT_DIR/architecture/deps.png"
fi
```

### 步骤 3：模块分析（阶段二）

```bash
# 识别核心模块
echo "## Identifying core modules..."

# 渲染系统
find "$ENGINE_PATH" -type d -name "*render*" -o -name "*rhi*" -o -name "*shader*"

# 资源管理
find "$ENGINE_PATH" -type d -name "*resource*" -o -name "*asset*" -o -name "*content*"

# ECS/场景
find "$ENGINE_PATH" -type d -name "*ecs*" -o -name "*scene*" -o -name "*entity*"

# 物理
find "$ENGINE_PATH" -type d -name "*physics*" -o -name "*collision*"

# 脚本
find "$ENGINE_PATH" -type d -name "*script*" -o -name "*lua*" -o -name "*python*"

# 音频
find "$ENGINE_PATH" -type d -name "*audio*" -o -name "*sound*"

# 为每个模块提取核心类
for module in rendering resources ecs physics scripting audio; do
  echo "### Analyzing $module module..."
  find "$ENGINE_PATH" -path "*/$module/*" -name "*.h" | head -20
  grep -r "class.*:" --include="*.h" "$ENGINE_PATH/$module" | head -20
done
```

### 步骤 4：版本历史分析（阶段三）

```bash
if [ "$VCS" = "git" ]; then
  echo "## Analyzing Git history..."

  # 提交历史
  git -C "$ENGINE_PATH" log --oneline --graph --all -n 100 > "$OUTPUT_DIR/changelog/history.txt"

  # 作者统计
  git -C "$ENGINE_PATH" shortlog -sn --all > "$OUTPUT_DIR/changelog/contributors.txt"

  # 标签和版本
  git -C "$ENGINE_PATH" tag -l | sort -V > "$OUTPUT_DIR/changelog/versions.txt"

  # 模块演进
  for module in rendering physics; do
    git -C "$ENGINE_PATH" log --oneline --name-only -- "$module" > "$OUTPUT_DIR/changelog/$module-evolution.txt"
  done
fi
```

### 步骤 5：生成文档（阶段四）

```bash
# 生成主文档
cat > "$OUTPUT_DIR/README.md" << 'EOF'
# Engine Analysis Documentation

> Generated by ge-module-analysis on $(date)

## Quick Links

- [Architecture Overview](architecture/README.md)
- [Module Documentation](modules/)
- [Developer Guides](guides/)
- [API Reference](api-reference/)
- [Changelog](changelog/)
- [Glossary](glossary.md)

## Engine Information

- **Engine**: $ENGINE_NAME
- **Source Path**: $ENGINE_PATH
- **Language**: $LANG
- **Version Control**: $VCS

EOF

# 生成术语表
cat > "$OUTPUT_DIR/glossary.md" << 'EOF'
# Engine Terminology

## Core Concepts

| Term | Definition | Related Concepts |
|------|------------|------------------|
| RHI | Render Hardware Interface | Vulkan, DirectX, OpenGL |
| ECS | Entity-Component-System | Entity, Component, System |
| ... | ... | ... |

EOF
```

## 高级用法

### 自定义分析范围

```bash
# 只分析渲染系统
/ge-module-analysis --focus rendering

# 只执行架构分析和模块分析
/ge-module-analysis --phases 1,2

# 分析特定版本
/ge-module-analysis --version 5.3 --path /root/UE5.3
```

### 集成到 CI/CD

```yaml
# .github/workflows/engine-analysis.yml
name: Engine Analysis

on:
  push:
    branches: [main]

jobs:
  analyze:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Run Module Analysis
        run: |
          /ge-module-analysis \
            --engine UnrealEngine \
            --path ./Engine \
            --output ./docs

      - name: Upload Documentation
        uses: actions/upload-artifact@v3
        with:
          name: engine-docs
          path: docs/
```

## 常见问题

### Q: 分析大型引擎时内存不足？

**A**: 分阶段执行，限制并发：
```bash
/ge-module-analysis --phases 1  # 先执行架构分析
/ge-module-analysis --phases 2 --focus rendering  # 再分析渲染模块
```

### Q: 如何分析非标准目录结构？

**A**: 使用 `--focus` 指定模块路径：
```bash
/ge-module-analysis --focus ./custom-modules/render,./custom-modules/physics
```

### Q: 生成的文档太大？

**A**: 使用 `--focus` 只分析关键模块：
```bash
/ge-module-analysis --focus rendering,resources
```
