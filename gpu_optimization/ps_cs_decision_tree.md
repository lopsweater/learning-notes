# PS vs CS 决策树（可视化版）

```
                    ┌─────────────────────────────┐
                    │      你的任务是什么？         │
                    └─────────────┬───────────────┘
                                  │
                    ┌─────────────┴─────────────┐
                    ▼                           ▼
        ┌───────────────────────┐   ┌───────────────────────┐
        │  需要 图形输出？        │   │  只需要 计算数据？      │

                    │                           │

                    │                           │
```

## PS 决策子树

```
                    ┌─────────────────────────────┐
                    │       图形输出任务           │
                    └─────────────┬───────────────┘
                                  │
            ┌─────────────────────┼─────────────────────┐
            │                     │                     │
            ▼                     ▼                     ▼
   ┌────────────────┐   ┌────────────────┐   ┌────────────────┐
   │ 需要混合/深度？  │   │ 大量纹理采样？  │   │ 滤波/模糊？     │
   └────────┬───────┘   └────────┬───────┘   └────────┬───────┘
            │                     │                     │
            ▼                     ▼                     ▼
     ┌─────────────┐       ┌─────────────┐       ┌─────────────┐
     │   ✓ 必须 PS  │       │   ✓ 必须 PS  │       │   ✓ 必须 PS  │
     │   (硬件优势)  │       │   (TMU缓存)  │       │   (Tile缓存) │
     └─────────────┘       └─────────────┘       └─────────────┘
```

### PS 最佳场景（速度优势）

```
┌────────────────────────────────────────────────────────────┐
│                    像素着色器 (PS) 完胜                      │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  1. 滤波/模糊类                                            │
│     ├── Bloom、高斯模糊                                     │
│     ├── 阴影模糊、AO 滤波                                   │
│     └── 优势：Tile缓存 + TMU采样，快 3~8 倍                 │
│                                                            │
│  2. 简单后处理                                             │
│     ├── ToneMapping、ColorGrade                            │
│     ├── FXAA/TAA、LUT                                      │
│     └── 优势：片上缓存友好，快 1.5~3 倍                     │
│                                                            │
│  3. 粒子/透明渲染                                          │
│     ├── 粒子系统、透明物体                                  │
│     ├── 需要混合、深度测试                                  │
│     └── 优势：硬件混合 + Early-Z，快 5~20 倍               │
│                                                            │
│  4. 不透明物体渲染                                         │
│     ├── 几何体、地形、建筑                                  │
│     └── 优势：硬件光栅化 + HiZ，无法替代                    │
│                                                            │
│  5. 光线 marching / SSR                                    │
│     ├── 复杂采样、光线步进                                  │
│     └── 优势：采样密集型，略快或持平                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## CS 决策子树

```
                    ┌─────────────────────────────┐
                    │       计算数据任务           │
                    └─────────────┬───────────────┘
                                  │
            ┌─────────────────────┼─────────────────────┐
            │                     │                     │
            ▼                     ▼                     ▼
   ┌────────────────┐   ┌────────────────┐   ┌────────────────┐
   │ 需要共享内存？  │   │ 需要原子操作？  │   │ 可异步并行？    │
   └────────┬───────┘   └────────┬───────┘   └────────┬───────┘
            │                     │                     │
            ▼                     ▼                     ▼
     ┌─────────────┐       ┌─────────────┐       ┌─────────────┐
     │   ✓ 必须 CS  │       │   ✓ 必须 CS  │       │   ✓ 必须 CS  │
     │   (SharedMem)│       │   (Atomics)  │       │   (并行吞吐)  │
     └─────────────┘       └─────────────┘       └─────────────┘
```

### CS 最佳场景（功能优势）

```
┌────────────────────────────────────────────────────────────┐
│                    计算着色器 (CS) 完胜                      │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  1. 数据更新类                                             │
│     ├── 粒子更新（位置、速度、生命周期）                     │
│     ├── 物理模拟                                           │
│     ├── 骨骼动画计算                                       │
│     └── 优势：Shared Memory + 高效写回                     │
│                                                            │
│  2. 结构化数据处理                                         │
│     ├── 视锥剔除、聚类                                     │
│     ├── 分桶、排序、前缀和                                 │
│     ├── 生成 IndirectArgs                                  │
│     └── 优势：原子计数 + 紧凑写入                          │
│                                                            │
│  3. 纹理/资源生成                                         │
│     ├── 程序化纹理生成                                     │
│     ├── 高度图更新                                         │
│     ├── 纹理压缩/解压                                      │
│     └── 优势：写密集型更适合 UAV                          │
│                                                            │
│  4. 异步计算                                               │
│     ├── 可与图形管线并行                                   │
│     ├── 不依赖帧同步                                       │
│     └── 优势：提高总吞吐量，而非单任务速度                 │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 关键原则总结

```
┌────────────────────────────────────────────────────────────┐
│                     一句话总结                               │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  PS 的优势：同任务更快（Tile缓存 + TMU + 硬件管线）         │
│  CS 的优势：总吞吐更大（异步 + 并行 + 灵活）                │
│                                                            │
│  选择标准：                                                 │
│  • 采样密集 + 图形输出 → PS                                │
│  • 计算密集 + 数据更新 → CS                                │
│  • 需要混合/深度 → PS                                      │
│  • 需要原子/排序 → CS                                      │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 参考：GPU 硬件差异

```
┌────────────────────────────────────────────────────────────┐
│                    PS 硬件优势                              │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  1. Tile-Based Rendering                                   │
│     └── 片上缓存，减少内存带宽                              │
│                                                            │
│  2. Texture Mapping Unit (TMU)                            │
│     └── 专用采样硬件，延迟更低                              │
│                                                            │
│  3. Raster Operations (ROP)                               │
│     └── 硬件混合、深度测试、MSAA                           │
│                                                            │
│  4. Early-Z / Hi-Z                                        │
│     └── 提前剔除，节省计算                                  │
│                                                            │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│                    CS 硬件优势                              │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  1. Shared Memory (LDS)                                   │
│     └── 线程组内高速共享内存                                │
│                                                            │
│  2. Atomic Operations                                      │
│     └── 高效原子计数、比较交换                              │
│                                                            │
│  3. Wave/Warp Operations                                   │
│     └── 线程束同步、广播、投票                              │
│                                                            │
│  4. Async Compute Queue                                   │
│     └── 与图形队列并行执行                                  │
│                                                            │
└────────────────────────────────────────────────────────────┘
```
